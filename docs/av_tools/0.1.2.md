## [v0.1.2] - 2026-04-07

### 新增功能 (Added)
* 新增调试工具 `tools/av_tools/debug_web_fetch.py`，支持命令行抓取网页 HTML 并生成解析 JSON 报告。
* 新增调试工具模块化目录 `tools/av_tools/debug_web_fetch_lib/`，按 `cli`、`fetchers`、`parsers`、`utils`、`models` 拆分职责，便于后续扩展站点规则。
* 新增诊断脚本 `temp/diagnose_code_count_mismatch.py`，用于对比目录扫描与 `codes.json` 的编号统计差异并输出原因样例。

### 技术改进/重构 (Changed/Refactor)
* 重构 `tools/av_tools/debug_web_fetch.py` 为薄入口，保持原命令不变并转发到模块化实现。
* 调整 `javbus_browser` 番号识别逻辑，补充无码日期型编号与厂商后缀变体归一能力。

### 修复 (Fixed)
* 修复 `fetch-metadata` 对边缘厂商编号的查询清洗问题：`031315-827-carib` 自动归一为 `031315-827`，`091416_382-1pondo` 自动归一为 `091416_382`。
* 修复 Provider 候选匹配对特殊厂商编号的等价识别，避免因后缀差异导致误判 `NOT_FOUND`。
* 修复并补充 `HEYZO` 与 `TOKYO-HOT` 特殊厂商 canonical 编号处理说明与测试，防止后续维护误删规则。
* 修复 `javbus_browser` 对无码日期番号的检索路径，`031315-827` / `052716-172` / `091416_382` 优先走 `/uncensored/search/`，避免在 `/search/` 下误报 `NOT_FOUND`。
